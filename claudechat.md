## IDENTITY

- **Name**: Quayside
- **Type**: UK fish auction price aggregation platform
- **Stack**: Python 3.9+ / Flask / SQLite / Jinja2
- **Status**: Live in production at https://quaysidedata.duckdns.org/
- **Purpose**: Scrapes daily fish price and landing data from public sources across UK ports, normalizes into a single database, generates daily digest reports, and provides web dashboards for port operators, fish traders, and the public. Also supports a port upload workflow where ports email or upload price sheets that are AI-extracted and human-approved before publishing.

## BUSINESS LOGIC

### Scraping Pipeline

1. `run.py main()` orchestrates: import all scrapers тЖТ call each via `_run_scraper()` wrapper
2. Each scraper fetches from its source (SWFPA XLS, PDF, HTML table, SSA XLSX)
3. Returns `list[PriceRecord]` тАФ no raw dicts ever
4. `_run_scraper()` catches all exceptions тЖТ returns `(results, error_info)` тАФ one port failing doesn't crash the pipeline
5. Results are upserted to DB via `db.upsert_prices()`
6. Per-port CSV exported to `output/prices_{port}_{date}.csv`
7. HTML digest generated via `report.generate_report(date)`
8. Email digest sent if SMTP configured

### ETag-aware Update Mode

1. `run.py update_run()` for intraday checks
2. Fetches SWFPA event links for today
3. Uses `http_cache.cached_fetch()` which checks ETag тЖТ Last-Modified тЖТ content MD5
4. Only processes files where `is_new=True`
5. Cache stored in `data/http_etag_cache.json`

### Species Normalisation

1. Raw names stored in DB exactly as scraped (e.g., "Monks", "HADDOCK", "Monk Or Anglers")
2. `species.normalise_species(raw)` maps to canonical name (e.g., "Monkfish") via `_CANONICAL_MAP`
3. 90+ canonical names with multiple raw aliases each
4. Unmapped names are title-cased as fallback
5. `is_noisy_species()` filters out: standalone noise words (offal, roe, mixed), substrings (damage, liver), Brixham concatenated suffixes (Dam, Bru, etc.)
6. `KEY_SPECIES`: 10 benchmark species for ticker and landing page (Haddock, Cod, Monkfish, Hake, Lemon Sole, Megrim, Whiting, Plaice, Turbot, Squid)
7. `_CATEGORY_MAP`: assigns species to demersal, flatfish, shellfish, pelagic, other

### Quality Checks (11 total)

1. **Outlier prices** тАФ Median + MAD (Median Absolute Deviation), robust z-score > 3.5├ЧMAD
2. **Low record count** тАФ Today's count <40% of rolling median for that port
3. **Stale data** тАФ No data for >=2 trading days (warn) / >=4 trading days (error)
4. **Daily avg spike** тАФ Port daily average ┬▒50% (warn) / ┬▒100% (error) from rolling average
5. **Seeded data** тАФ Guards against known seeded timestamps on live ports
6. **Live site smoke test** тАФ Fetches each port dashboard URL and verifies displayed price matches DB
7. **Unknown fields** тАФ NULL or "Unknown" values in required fields
8. **Unmapped species** тАФ Species not in `_CANONICAL_MAP`
9. **Price sanity** тАФ <=0, >┬г200/kg, or price_low > price_high
10. **Date sanity** тАФ Future dates or dates >1 year old
11. **Price swings** тАФ >200% (warn) / >500% (error) vs 30-day mean

## CONTEXT FOR AI

### Critical Rules

1. **Demo Port isolation** тАФ NEVER include Demo Port in real data views. It uses synthetic data in a separate `demo_prices` table. Filter by `data_method != 'demo'` and `slug != 'demo'`. This keeps getting accidentally re-introduced.

2. **Single worker** тАФ Gunicorn must run with `--workers 1` because APScheduler runs in-process and duplicates would cause double scraping.

3. **Species are normalised at display time** тАФ Store raw names in DB, map to canonical at render time. Never normalize before storage.

4. **Editorial design aesthetic** тАФ The brand is "newspaper meets financial data". Sharp corners (2px max radius), borders not shadows, monospace for data, serif italic for species names, warm muted colour palette.

### Decisions Made

- SQLite over Postgres: simplicity for single-server deployment, WAL mode for concurrency
- Blueprint refactoring: moved from monolithic app.py to 8 blueprint files for maintainability
- SWFPA as primary source: most UK ports publish through SWFPA event pages
- AI extraction as fallback: Claude API handles unknown file formats automatically
- Auto-publish after 2 hours: balances data freshness vs manual review
- ETag caching: avoids re-processing unchanged files during intraday updates

- macOS launchd scheduling: replaced with systemd timers on production server
