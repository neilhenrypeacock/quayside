Quayside Data

The UK fish market,
in one place.

Fish auction price data in the UK is scattered across port websites, PDFs, spreadsheets, and HTML tables — each in a different format, updated at different times, with different species names. Quayside collects it, normalises it, and makes it comparable.

The problem we’re solving

A fish buyer in Brixham wanting to know whether cod is cheaper at Peterhead today has no straightforward way to find out. Port websites are inconsistent. Some publish PDFs. Some publish HTML tables. Some don’t publish at all. Species names differ between ports — what Peterhead calls “Monks”, Brixham may call “Monkfish”. Dates are formatted differently. Grades are incomparable.

Quayside solves this by running daily scrapers against each port’s published data, normalising species names and grades into a common schema, and storing everything in a single database. The result is a clean, queryable feed of UK fish market prices — updated automatically every day.

We don’t hold any data that isn’t already public. We just make it usable.

How it works
Step 01 — Collect

Automated scrapers, daily

Each morning, port-specific scrapers fetch the latest price data from public sources. No manual downloading. No copy-paste.

  • PDF extraction via pdfplumber (Brixham, Newlyn)
  • HTML table parsing (Peterhead landings, Scrabster, Lerwick)
  • XLS file ingestion (Peterhead prices via SWFPA)
  • One failing port doesn’t stop the others
Step 02 — Normalise

One schema, every port

Raw data is mapped to a common structure before it reaches the database. Species names, dates, and grades are all standardised.

  • Species aliases resolved to canonical names (e.g. “Monks” → “Monkfish”)
  • Dates stored as ISO 8601 (YYYY-MM-DD)
  • Prices in £/kg, grades noted as provided
  • Upsert strategy: latest scrape wins
Step 03 — Publish

Data where you need it

Normalised data is surfaced across several interfaces, updated daily. No login required to browse market data.

  • Port dashboards with cross-market benchmarks
  • Daily, weekly, and monthly digest reports
  • CSV export per port
  • JSON API (in development)
